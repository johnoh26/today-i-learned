- **OData** (Open Data Protocol) — combined with .NET Web API, lets you easily set up server-side queries
- Allows organizing and filtering data on a powerful server rather than pulling everything to the client
- Similar functionality to GraphQL but different in approach — easier to set up for .NET Web API (no schemas, resolvers, etc.)

**Reference:** [OData vs GraphQL vs ORDS](https://www.progress.com/blogs/rest-api-industry-debate-odata-vs-graphql-vs-ords)
